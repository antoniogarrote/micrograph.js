Micrograph(options, callback)
Builds a new Micrograph object.

Parameters:
{Object} options Optional
  • persistent: should the store use persistence?
  • treeOrder: in versions of the store backed by the native indexing system, the order of the BTree indices
  • name: when using persistence, the name for this store. In the MongoDB backed version, name of the DB used by the store. By default 'rdfstore_js' is used
  • overwrite: clears the persistent storage
  • maxCacheSize: if using persistence, maximum size of the index cache
{Function} callback Optional
Callback function that will be invoked with the initialised instance of the store.

from(uri, options, callback)
Creates a data selection from a remote data source using an AJAX+CORS or JSONP request.
Parameters:
{String} uri Optional
URI where the data will be retrieved.
{Object} options Optional
Hash of options for the remote request.
  • media: Media type to request and process. Possible values are: microdata and n3. If none is specifed JSON will be requested.
  • jsonp: If a JSONP request must be used, name of the callback or true for a name to be randomly generaed
  • crossDomain: will force the use of XDomainRequest object in IE browsers not supporting CORS
  • callback: Name of the callback parameter for the JSONP request.
  • timeout: Max. number of milliseconds before the JSONP request times out.
  • maxRetries: Max. number of times the JSONP request will be retrievd before giving up.
  • result: Optional name of property in a JSONP response where the actual answer data is stored.
{Function} callback Optional
Function that will be invoked when the data have been retrieved.
